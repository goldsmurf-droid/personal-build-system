# Stack Defaults

Personal build defaults for Ryan Goldberg. These apply to every personal project unless the spec explicitly states otherwise. A deviation in the spec is a deliberate choice, not an oversight.

## Language

- **Python 3.11+** — default for everything unless there's a specific reason not to.
- Type hints on all function signatures.
- No unnecessary OOP. Functions and modules first; classes when they're the right tool.

## Runtime / hosting

- **Local-first.** Runs on dev machine. No deploy step unless the problem requires remote access or always-on behavior.
- **Tier 1 (VM):** Named host in `infra-defaults.md`. Deploy via git pull + systemd unit or simple process manager. No containers unless complexity justifies it.
- **Tier 2 (cloud):** Deliberate choice only. Explicitly stated in spec.

## Source control

- One repo per project.
- GitHub or Bitbucket — whichever is already in use for related work.
- `main` branch only until the project is complex enough to need branches.
- Commit the spec first, always.

## Dependencies

- Prefer stdlib. Add a dependency only when the stdlib alternative is materially worse.
- `requirements.txt` for all projects. Pinned versions.
- No dependency injection frameworks. No ORM unless the data model is complex enough to need one.

## State / persistence

- **File-based until it stops working.** JSON or JSONL for structured data. Plain text or Markdown for human-readable state. SQLite when relational queries become necessary. Never start with a database.
- State files live in a `data/` directory within the project, or in a user-specified path passed at invocation.

## Auth

- No auth unless the problem requires it.
- If required: simplest possible. API key in environment variable first. OAuth only when the target service requires it.
- No rolling your own auth. Ever.

## Testing

- **Smoke tests in pytest.** Cover the happy path and the top 2-3 failure modes. If it passes smoke tests, it's shippable.
- **Calibration tests for AI-shaped components.** Any component that calls an LLM gets a calibration test: a fixed input with a known acceptable output range. Not a unit test — a sanity check.
- No mocking unless I/O is unavoidable to test. Test against real behavior where possible.
- Test file: `tests/test_smoke.py`. Run with `pytest tests/`.

## Documentation

- **Markdown for everything human-readable.**
- `SPEC.md` — source of truth. Lives in project root.
- `README.md` — generated by Claude Code. How to install and run.
- `run.md` — generated by Claude Code. How to run from a fresh pull on any machine.
- `claude.md` — Claude Code's working log. Assumptions, decisions, things skipped.
- No docstrings required on internal functions. Docstrings on public interfaces only.

## Code style

- Black for formatting. No exceptions.
- Ruff for linting.
- No type: ignore comments without an explanatory note.
- Flat is better than nested. If something needs more than two levels of indentation, it probably needs a function.

## CLI conventions

- `argparse` for CLI argument parsing.
- `--help` always works.
- Exit codes: 0 for success, 1 for user error, 2 for system error.
- Stderr for errors and logs. Stdout for output only.

## Environment

- `.env` file for secrets and config. Never committed to git.
- `.env.example` committed with all keys, no values.
- `python-dotenv` for loading `.env`.

## What "done" means

- Smoke tests pass.
- `README.md` exists and is accurate.
- `run.md` exists and works from a fresh clone.
- No secrets in git history.
- Spec and built thing are consistent.
