## Secrets handling

- Secrets live in .env at project root. Never committed.
- .env.example committed with all keys, no values.
- .env placed manually on VM after fresh clone.
- No secrets manager unless a project explicitly requires one.

## Deploy pattern

All projects run on personal-vm. Default deploy:

    cd ~/projects/active/<slug>
    git pull
    python3.12 -m pip install -r requirements.txt
    # if running as a service:
    sudo systemctl restart <slug>

Projects that run as a service get a systemd unit file generated by Claude Code.
